# HAKOMONO 仕様書

最終更新: 2025-11-30

## 1. 概要
- プロジェクト名: HAKOMONO
- 種別: 社内向けデータ集計・出力ツール（Streamlit アプリケーション）
- 目的: 受注データ（Excel）を読み取り、社内所定のテンプレート（Excel マクロ有効ファイル `.xlsm` 等）へ加工・出力することで集計業務を自動化する。

## 2. 対象読者
- 開発者、運用担当者、QA、将来的な外部レビュー担当。

## 3. 用語定義
- ソースファイル: ユーザーがアップロードする受注データ（`.xlsx` / `.xlsm`）。
- テンプレート: 集計先テンプレートファイル（`.xlsm`）。
- 出力ファイル: 生成される集計結果（`.xlsm`）。

## 4. ユースケース（主要）
- UC-01: ユーザーがソースファイルとテンプレートをアップロードし、集計ファイルをダウンロードする。
- UC-02: 不正な入力（列欠損や形式不一致）を検出し、ユーザーへエラーを表示する。
- UC-03: 運用者がローカルで Streamlit を起動してアプリを利用する。

## 5. 機能要件
5.1 入出力
- 入力: ソースファイル（`.xlsx`, `.xlsm`）およびテンプレート（`.xlsx`, `.xlsm`）を UI で受け付ける。
- 出力: `集計表YYMMDD.xlsm` の形式でユーザーにダウンロード提供する。

5.2 データ処理
- ソースから特定の条件（例: 'Box' アイテム）を抽出して、`Koda` と `Yamato` の集計に分割する。
- マッピング・ヘッダー整形を行い、テンプレートへ書き込み可能な形式（DataFrame 等）に変換する。

5.3 エラーハンドリング
- 入力ファイルの読み込み中にエラーが発生した場合はユーザーへエラーメッセージを表示し、処理を中断する。
- 取りうるエラー例: ファイル破損、必須列欠損、依存パッケージ未インストール、テンプレート書き込み失敗。

5.4 ロギング
- ローカル実行時は標準出力（ターミナル）にログを出力する。運用時にファイルへリダイレクトして保存可能とする。

## 6. 非機能要件
- 実行環境: Windows（開発環境）、Python 3.8 以上（動作確認は 3.12）
- ライブラリ: `streamlit`, `pandas`, `openpyxl` など（`requirements.txt` に記載）
- パフォーマンス: 中程度の Excel（数万行程度）を数十秒〜数分で処理可能であること（具体的な SLA は要計測）。
- セキュリティ: 機密情報は `.env` に格納し、`.gitignore` に追加してリポジトリに含めないこと。

## 7. 画面 / UI
- 単一ページ（Streamlit）で次の要素を提供:
  - ファイルアップロード（Source File, Template File）
  - 実行ボタン（Run Aggregation）
  - 実行中のインジケータ（spinner）
  - 結果ダウンロードボタン
  - 結果プレビュー（Koda / Yamato DataFrame の一部表示）

## 8. データフォーマット（想定）
- ソースファイル: Excel のテーブル形式（ヘッダー行を持つ）。必須カラム名は実装に依存するため、`TROUBLESHOOTING.md` によるエラー例を参照すること。
- テンプレート: 書き込み先のセル配置やマクロは既存のテンプレートに従う。

## 9. 運用手順
9.1 開発者ローカル起動
```powershell
cd C:\Users\win\for-python\hakomono
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
streamlit run app.py
```

9.2 サーバー起動・停止
- 手動: ターミナルで `streamlit run app.py` を実行／Ctrl+C で停止。
- 常駐化: Windows タスクスケジューラまたは NSSM 等でサービス化を検討する（別途ドキュメントを追加予定）。

9.3 トラブル時の対応
- ポート競合: `netstat -ano | Select-String ':8501'` で PID を確認し、不要なら `Stop-Process -Id <PID>` で停止する。ログは `streamlit_start.log` に保存して解析する。

## 10. 依存関係
